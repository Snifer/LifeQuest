# LifeQuest Roadmap

This roadmap is based on the feature backlog collected in issues.

It is organized by priority and intended release horizon, so it can evolve with real usage and feedback.

---

## Guiding priorities

1. Improve first-time user experience
2. Strengthen the core daily workflow
3. Make the dashboard and widgets more flexible
4. Deepen add-ons without making them mandatory
5. Keep the plugin maintainable and local-first

---

## v1.0.2 — First usability and publishing improvements

Focus: reduce first-run friction, improve release readiness, and strengthen the most visible user flows.

### Planned features

- Expand first-run onboarding and guided setup
  - onboarding after activation
  - choose language, hero class, life areas, and add-ons
  - avoid dropping users directly into advanced settings

- Improve publication assets and user-facing documentation
  - screenshots and GIFs in README
  - clearer privacy/local-first section
  - better quick-start instructions
  - more release-oriented changelog structure

- Improve daily note block customization
  - more configurable generated block
  - group quests by area
  - insert only pending quests
  - improve sync after manual edits

## v1.0.3 - Internationaization structure

- Improve internationalization structure for future languages
  - move more hardcoded strings into i18n
  - review visible defaults and labels
  - prepare structure for more languages

### Goal
Make the plugin easier to understand, easier to install, and more presentation-ready for community adoption.

---

## v1.1.0 — Core workflow improvements

Focus: improve the daily execution loop and make the plugin more useful inside notes.

### Planned features



- Add more powerful widget filters and summary widgets
  - filters by area, period, and status
  - more consistent compact/card variants
  - next quest widget
  - top priorities widget
  - richer weekly summary widget

- Improve weekly review insights and recommendations
  - more actionable insights
  - previous-week comparison
  - detect neglected or overloaded areas
  - suggest difficulty adjustments

### Goal
Strengthen the everyday loop: plan, act, review, and reflect.

---

## v1.2.0 — Configurable workspace and safer data flows

Focus: let users shape the plugin to their workflow and trust their data more.

### Planned features

- Make dashboard layout modular and configurable
  - reorder sections
  - hide or lock unused panels
  - custom layout preferences
  - configurable quick actions

- Add import, backup, and stronger data migration tools
  - JSON import
  - backup and restore from settings
  - stronger validation for older data
  - clearer versioned migrations

### Goal
Make the plugin more customizable and safer for long-term use.

---

## v1.3.0 — Add-on expansion

Focus: deepen optional systems while keeping the core plugin lightweight.

### Planned features

- Expand Hero Shop add-on functionality
  - filterable reward categories
  - clearer redemption history
  - per-reward limits and cooldowns
  - better repeatable vs one-time visibility
  - improved coin economy balance

- Strengthen health add-on with better progress tracking
  - better progress charts
  - weight goals
  - weekly and monthly trends
  - clearer reminders
  - richer exports

### Goal
Make optional modules feel complete and valuable without forcing them on every user.

---

## v1.4.0 — Advanced planning systems

Focus: support more advanced personal planning and habit design.

### Planned features

- Add advanced quest system capabilities
  - subtasks
  - due dates
  - chained quests
  - more flexible recurring quests
  - smart pause without penalty

### Goal
Support more complex workflows while preserving the simplicity of the base system.

---

## v2.0.0 — Long-term architecture and scale

Focus: stabilize the plugin as it grows in scope and usage.

### Planned features

- Reduce internal technical debt and improve maintainability
  - split large files further
  - add more tests for widgets, settings, and add-ons
  - improve automatic version migrations
  - reduce UI/store coupling

### Goal
Prepare LifeQuest for long-term evolution without slowing down feature development.

---

## Notes

- Versions in this roadmap are planning targets, not fixed promises.
- Priority may change based on user feedback, publishing needs, and maintenance cost.
- Add-ons should remain optional wherever possible.
